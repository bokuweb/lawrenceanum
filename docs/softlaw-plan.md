# 検討メモ: 通達・ガイドライン (soft law) の収集

> 法令本文(e-Gov)に載らない行政の規範=通達・ガイドライン・告示・Q&A を収集する計画。
> 2026-06-19 調査。関連: 既存の官報収集（告示はそちらでカバー）。

## 結論
- 横断ポータルは**無い**（e-Gov法令APIは硬法令のみ、告示・通達・ガイドラインは対象外）。省庁個別。
- 着手は **国税庁 法令解釈通達** が最有力＝実装済みMVP（`tsutatsu-client`）。理由:
  - 静的HTML・robots `/law/` 許容・デジタル庁「公共データ利用規約(PDL1.0=CC BY4.0互換)」で商用/再配布可。
  - 「通達」は**著作権法13条2号で著作権なし**（二重にクリーン）。税理士セグメントにも刺さる。
- 次点: **個人情報保護委員会ガイドライン**（各編が「告示」として発出→13条非保護、URL安定）。
  金融庁 監督指針/事務ガイドライン（PDL1.0だがPDF比率高くコスト大）。

## 著作権13条の線引き
- 自由利用: 通達本文・告示として発出されたガイドライン（個情委等）。
- グレー(要利用規約依拠/法務確認): タックスアンサー等のQ&A・解説（創作的記述は保護され得る）。
- 編集著作物リスク: 各省サイトの一覧/分類構造を丸ごと複製しない。個別文書を自社で再構成する設計。

## 官報収集との関係
- 重複: 告示（官報で既収集）。
- 補完(新規価値): 通達・事務ガイドライン・監督指針・Q&A は官報に載らず各省サイトのみ。

## 実装状況 / 次
- ✅ `tsutatsu-client`（国税庁・所得税基本通達）MVP: 目次→本文ページ→通達項目(番号/見出し/本文)抽出。
  CLI `tsutatsu-fetch`/`tsutatsu-build-json` → `public/tsutatsu/{tax}.json`。ライブ検証済(73頁)。
- 次: (a) 他税目(法人税/相続税/消費税)へ拡張、(b) フロント「通達」ビュー＋検索索引取り込み、
  (c) 法令↔通達クロスリンク（通達本文の「法第N条」参照）、(d) CI接続、(e) 個情委ガイドライン。
